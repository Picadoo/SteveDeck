## 1. ⚠️ 构建必读（绕坑，否则寸步难行）
- **pnpm 脚本运行器会卡死**：`pnpm run X` / `pnpm -C ... build|typecheck` 触发 `verify-deps-before-run` 预检（陈旧 `.modules.yaml`）→ `install --production` 无 TTY 中止。**直接调工具**：
  - engine typecheck：`node packages/engine/node_modules/typescript/bin/tsc -p packages/engine/tsconfig.json --noEmit`（`allowJs` 会校验 .js 语法；`checkJs:false` 不类型检查 .js）
  - engine 单测：`node --test packages/engine/test/auto_use.test.cjs`
  - engine e2e：`node packages/engine/test/control-plane.cjs`（**起全新引擎实例、跑当前码 → 引擎侧最可靠的验证**）
  - ui typecheck：`node packages/ui/node_modules/typescript/bin/tsc -p packages/ui/tsconfig.json --noEmit`
  - ui 构建：`cd packages/ui` 后 `node node_modules/vite/bin/vite.js build`（**必须 cwd=packages/ui**，否则找不到 index.html）
  - protocol 构建：`cd packages/protocol` 后 `& .\node_modules\.bin\tsup.cmd`（改了协议类型后必跑，engine/ui 才看得到新类型）。protocol 有 CJS 构建，引擎 `.js` 可 `require("@mcbot/protocol")`。
  - cargo 不在 PATH：先 `$env:PATH="$($env:USERPROFILE)\.cargo\bin;$env:PATH"`，再 `cargo check --manifest-path apps/desktop/src-tauri/Cargo.toml`
- **预览引擎可能是旧实例（踩过 3 次！）**：浏览器预览（vite preview, port 5273, serverId 见 `preview_list`）连的常驻引擎进程**不一定热重载最新引擎源码**。改了引擎（新模块/命令/handler）后，在预览里跑「UI→引擎」往返常超时或停在旧码（开关接受但默认规则不加载、新命令无响应）。**别对着旧预览引擎反复调**——引擎侧靠 `control-plane.cjs`+单测验证；要在预览里看活的，先**重启 dev 引擎**（会短暂断 Picado，自动重连）。preview_screenshot 会超时，用 `preview_eval` 读 DOM。
- **安装包**：见 `.claude` 记忆 `project_desktop_build` 或下方 §4。
